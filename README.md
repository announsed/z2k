## ⚡ Краткое описание

**z2k** — это модульный установщик **zapret2** для роутеров Keenetic с Entware, который автоматизирует:

- ✅ Установку zapret2 без компиляции ( готовые бинарники openwrt-embedded)
- ✅ Настройку **autocircular стратегий** с автоподбором под вашего провайдера
- ✅ Поддержку **IPv4/IPv6** с автоматическим определением доступности
- ✅ Прозрачное проксирование **Telegram** через MTProto
- ✅ Обход блокировок для **YouTube** (TCP + QUIC/HTTP3), **Discord**, реестра РКН

> **💡 Важно:** После установки autocircular стратегиям требуется время (от нескольких секунд до минут) для подбора рабочих параметров. Если сайт не открывается сразу — дайте странице 2-3 раза перезагрузиться. Система автоматически переберёт стратегии и запомнит рабочую.

## ✨ Возможности

### 🔹 Обход блокировок

| Протокол | Профиль | Стратегий | Описание |
|----------|---------|-----------|----------|
| **TCP** | RKN | 45 | Заблокированные сайты из реестра РКН |
| **TCP** | YouTube | 22 | youtube.com и связанные домены |
| **TCP** | YouTube GV | 22 | GoogleVideo CDN (стриминг видео) |
| **QUIC/UDP** | YouTube QUIC | 12 | YouTube over HTTP/3 (порт 443) |
| **TCP** | Discord | от RKN | Discord с отдельным списком доменов |
| **UDP** | Discord Voice/Video | circular_locked | Голосовые и видеозвонки с сохранением стратегии |

### 🔹 Автоматизация

- 🔄 **Autohostlist** — автоматическое применение стратегий только к доменам из списка (не влияет на остальной трафик)
- 🌐 **IPv6-ready** — автоопределение и настройка правил для IPv6 (если доступен)
- 📦 **Списки доменов** — загружаются автоматически из актуальных источников (zapret4rocket)
- 🧹 **Очистка бэкапов** — автоматическое удаление старых резервных копий конфигов

### 🔹 Telegram Proxy (MTProto)

- 🕵️ **Автоматический парсинг** прокси с публичных источников
- ⚡ **Проверка скорости** каждого прокси перед использованием
- 🎯 **Выбор лучшего** и обновление конфигурации Xray/XKeen
- ⏰ **По расписанию** — обновление каждые 2 часа через cron
- 🔒 **Прозрачно для клиентов** — не требует настройки на устройствах

## 📖 Подробная установка

### Шаг 1: Подготовка прошивки

Зайдите в веб-интерфейс Keenetic (`http://my.keenetic.net`):

1. Перейдите в **Управление** → **Параметры системы**
2. Установите компоненты:
   - ☑️ **Протокол IPv6**
   - ☑️ **Модули ядра подсистемы Netfilter**
3. Нажмите **Сохранить** и дождитесь перезагрузки

### Шаг 2: Настройка Entware

1. Подключите USB-накопитель (минимум 1 ГБ)
2. Установите компонент **"Операционная система Entware"**
3. Дождитесь установки (5-10 минут)
4. Подключитесь по SSH:
   ```bash
   ssh root@my.keenetic.net
   # пароль: admin (или ваш)
   ```

### Шаг 3: Обновление пакетов

```bash
# Обновить списки пакетов
opkg update

# Установить базовые зависимости
opkg install coreutils-sort curl grep gzip ipset kmod_ndms

# Опционально: cron для автообновления
opkg install cron
```

### Шаг 4: Запуск установщика

```bash
# Скачать и запустить установщик
curl -fsSL https://raw.githubusercontent.com/necronicle/z2k/master/z2k.sh | sh
```

Или вручную:

```bash
# Скачать скрипт
curl -fsSL https://raw.githubusercontent.com/necronicle/z2k/master/z2k.sh -o /tmp/z2k.sh

# Запустить
sh /tmp/z2k.sh
```

### Шаг 5: Проверка работы

```bash
# Статус сервиса
/opt/etc/init.d/S99zapret2 status

# Проверка процессов
ps | grep nfqws2

# Проверка iptables правил
iptables -t mangle -L -n -v | grep -i zapret
```

## 🔄 Как работает autocircular

### Принцип действия

1. **Набор стратегий**: Каждый профиль (RKN, YouTube, QUIC) содержит N стратегий с параметрами `--dpi-desync`, `--dpi-desync-split`, `--dpi-desync-fake-tls` и т.д.

2. **Мониторинг соединений**: Модуль `circular` в `nfqws2` отслеживает успех/неудачу TCP handshake для каждого домена.

3. **Переключение при неудаче**: После `fails=3` последовательных неудач домен автоматически переключается на следующую стратегию `(strategy+1) % total`.

4. **Запоминание успешной**: Успешная стратегия закрепляется за доменом до следующей серии неудач.

5. **Locked режим (Discord UDP)**: Для UDP-трафика (Discord voice/video) используется `circular_locked` — успешная стратегия сохраняется в файл `locked.tsv` и **не сбрасывается** при перезапуске сервиса.

### Пример конфигурации

```ini
# Профиль YouTube TCP (autocircular)
--dpi-desync=fake --dpi-desync-split=2 --dpi-desync-split-pos=1:MSS:2:MSS
--dpi-desync-fake-tls=tls_clienthello_www_google_com.bin
--hostlist=/opt/zapret2/ipset/yt_hostlist.txt
--autocircular=1:22:fails=3
```

Где:
- `1:22` — использовать стратегии с 1 по 22
- `fails=3` — переключать после 3 неудач
- `hostlist` — применять только к доменам из списка

### Время подбора

| Сценарий | Время подбора | Комментарий |
|----------|---------------|-------------|
| Первая установка | 30 сек — 5 мин | Зависит от количества доменов |
| После смены провайдера | 1-3 мин | Старые стратегии могут не работать |
| После обновления списков | 10-60 сек | Новые домены требуют подбора |
| Стабильная работа | 0 сек | Стратегии уже подобраны |

## 📱 Telegram Proxy

### Что это?

Модуль **автоматического проксирования Telegram** через MTProto. Работает прозрачно для всех устройств в сети — **не требует настройки на телефонах/компьютерах**.

### Как работает?

1. **Парсинг**: Скрипт загружает список MTProto прокси с публичного источника (`tg.html`)
2. **Проверка**: Каждый прокси тестируется на скорость и доступность
3. **Выбор**: Лучший прокси выбирается автоматически
4. **Настройка**: Конфигурация Xray обновляется с новым upstream-прокси
5. **Перезапуск**: Xray перезапускается для применения изменений
6. **Повтор**: Процесс повторяется каждые 2 часа через cron

### Файлы модуля

| Файл | Назначение |
|------|------------|
| `/opt/etc/zapret2/tg_proxy_settings.conf` | Настройки (включено/выключено, интервал) |
| `/opt/etc/zapret2/tg_proxy_secret` | Секретный ключ для локального MTProto сервера |
| `/tmp/tg_proxy_cache.txt` | Кэш распарсенных прокси |
| `/opt/var/log/tg_proxy.log` | Лог работы модуля |

### Управление

```bash
# Включить/выключить через меню z2k
sh z2k.sh menu → "Telegram Proxy" → Enable/Disable

# Запустить проверку вручную
/opt/etc/init.d/S99zapret2 tg-update

# Посмотреть лог
tail -f /opt/var/log/tg_proxy.log

# Проверить текущий прокси
grep "proxy" /opt/etc/xray/configs/05_mtproto.json
```

### Требования

- Установленный **Xray** или **XKeen** (устанавливается автоматически при включении функции)
- Свободно **~10 МБ** на разделе `/opt`
- Доступ к источнику прокси (`storage.yandexcloud.net`)

### Производительность

- **Парсинг 50 прокси**: ~0.02 сек (оптимизировано через `awk`)
- **Проверка одного прокси**: 2-5 сек (сокет + curl тест)
- **Общее время обновления**: 1-3 мин (в зависимости от количества рабочих прокси)

## 🐛 Диагностика

### Частые проблемы

#### 1. "Illegal instruction" при установке

**Причина**: Несоответствие архитектуры процессора и версии opkg/пакетов.

**Решение**:
```bash
# Проверить архитектуру
uname -m
opkg print-architecture

# Переустановить opkg для правильной архитектуры
# (следуйте инструкции Entware для вашей платформы)
```

#### 2. nfqws2 не запускается

**Причина**: Отсутствие модулей ядра или библиотек.

**Решение**:
```bash
# Проверить модули
lsmod | grep -E "xt_|nfnetlink"

# Проверить библиотеки
ldd /opt/zapret2/bin/nfqws2

# Переустановить зависимости
opkg install libnetfilter-queue libnfnetlink libmnl
```

#### 3. Сайты не открываются после установки

**Причина**: Autocircular ещё не подобрал рабочие стратегии.

**Решение**:
- Подождите 1-3 минуты
- Обновите страницу 3-5 раз
- Проверьте логи: `logread | grep nfqws2`

#### 4. Telegram не работает через proxy

**Причина**: Все прокси в списке нерабочие или Xray не запущен.

**Решение**:
```bash
# Проверить статус Xray
ps | grep xray

# Запустить обновление прокси вручную
/opt/etc/init.d/S99zapret2 tg-update

# Посмотреть лог
tail -50 /opt/var/log/tg_proxy.log
```

#### 5. IPv6 не работает

**Причина**: IPv6 отключён в прошивке или не настроен провайдером.

**Решение**:
- Включите компонент "Протокол IPv6" в веб-интерфейсе
- Проверьте наличие IPv6 адреса: `ifconfig br0 | grep inet6`
- z2k автоматически отключит IPv6 правила если протокол недоступен
